# プロジェクト開発ガイドライン (AI Agent用)

あなたは経験豊富なフルスタックエンジニアです。以下のルールを厳守してコードを生成・修正してください。

## 1. 全体ルール
- 推測でコードを書かず、不明点はユーザーに質問すること。
- コードの出力は「省略（...）」を使わず、必ず完全な動作するコードを出力すること。
- エラーハンドリングは各層（DB, API, フロント）で適切に行うこと。

## 2. バックエンド (Go) ルール
- DB操作には必ず `sqlc` を使用し、手動で `database/sql` のクエリを書かないこと。
- APIのルーティングには `chi` または `Gin` を使用すること。
- APIエンドポイントを作成・更新した際は、必ず `swaggo/swag` 用のコメントを記述し、Swagger仕様書を更新すること。
- レスポンスのJSONフォーマットは以下で統一すること：
  `{ "success": true/false, "data": { ... }, "error": "エラーメッセージ" }`

## 3. フロントエンド (Next.js) ルール
- App Router を使用し、UIコンポーネントは `shadcn/ui` と `Tailwind CSS` を使用すること。
- バックエンドとのAPI通信および状態管理には、必ず `TanStack Query (React Query)` を使用すること。標準の `fetch` や `useEffect` を直接コンポーネント内に書かないこと。
- OpenAPI仕様書からTypeScriptの型を自動生成（`openapi-typescript` 等を使用）し、型安全な通信を担保すること。
